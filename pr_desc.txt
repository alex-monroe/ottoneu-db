🚨 **Severity:** MEDIUM
💡 **Vulnerability:** Several API routes dealing with arbitration plans and surplus adjustments lacked explicit string length validation on user text inputs (like `name` and `notes`). This allowed arbitrarily large string payloads to be sent.
🎯 **Impact:** Extremely large text inputs could cause application memory exhaustion, database processing overhead, query timeouts, and lead to resource exhaustion / Denial of Service (DoS) attacks.
🔧 **Fix:** Added strict, explicit maximum length bounds (`maxLength` limits) to string fields in the API route handlers (`name` <= 100 characters, `notes` <= 2000 characters). Invalid inputs now fail early and securely return `400 Bad Request`.
✅ **Verification:** Verified by running the full test suite via `cd web && npm test` and running static analysis via `make check-arch` and `npm run lint`. Ensure test payloads conform to the new boundaries.
