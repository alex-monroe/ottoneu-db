💡 What: Wrapped the `filtered` list calculation in `PlayerSearch.tsx` in a `useMemo` hook, hoisted `query.toLowerCase()` outside the loop, and reorganized filter conditions.
🎯 Why: Prevents expensive `O(N)` list filtering on every component re-render when search properties don't change. Checking cheap enum conditions (`p.position === position`) before expensive string operations (`p.name.toLowerCase().includes(...)`) short-circuits the loop, saving computational cycles.
📊 Impact: Reduces computational load during re-renders, prevents redundant `toLowerCase` calls on large arrays, and ensures faster typing feedback when filtering.
🔬 Measurement: Verify changes in DevTools React Profiler when toggling positions vs. typing. Test suite confirms functionality remains perfectly identical.
