## 2024-03-12 - Enhance PlayerSearch empty state UX
**Learning:** Found a common UX pattern where empty states in searchable/filterable lists lack a direct action to reset the filters. Showing the query inside the empty state message provides helpful context, but automatically clearing all filters (query + category selectors) and refocusing the input with a single click significantly reduces friction for subsequent searches.
**Action:** When designing or refactoring searchable lists, ensure the empty state includes a one-click "Clear filters" action that resets both text and category filters, while keeping the user's keyboard focus within the primary search input.
## 2024-05-15 - Interactive UI indicators
**Learning:** For row expansion triggers in tables, using plain text characters like `▼` and `▶` is functionally sufficient but feels static and can be misinterpreted by screen readers if not paired with appropriate ARIA attributes. Replacing them with an animated standard UI icon (e.g., `ChevronRight` with a 90-degree rotate transition) combined with `aria-expanded` on the parent interactive row significantly improves both the visual feedback and the accessibility context of the interaction.
**Action:** When implementing expandable or togglable UI elements, always use scalable icons rather than text characters, ensure they are paired with a CSS transition for smooth state changes, and explicitly apply `aria-expanded` to the parent button/row.
