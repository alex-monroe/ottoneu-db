## 2024-03-12 - Enhance PlayerSearch empty state UX
**Learning:** Found a common UX pattern where empty states in searchable/filterable lists lack a direct action to reset the filters. Showing the query inside the empty state message provides helpful context, but automatically clearing all filters (query + category selectors) and refocusing the input with a single click significantly reduces friction for subsequent searches.
**Action:** When designing or refactoring searchable lists, ensure the empty state includes a one-click "Clear filters" action that resets both text and category filters, while keeping the user's keyboard focus within the primary search input.
## 2025-01-28 - Add loading spinner to async submit button
**Learning:** Adding a loading spinner next to button text during asynchronous operations provides immediate, clear visual feedback. This improves UX by assuring the user their action is being processed and preventing multiple accidental submissions, especially on critical actions like login or registration. Using an SVG spinner with the Tailwind `animate-spin` class is lightweight and integrates well with existing UI patterns. Ensure the SVG includes `aria-hidden="true"` so screen readers rely on the text change (e.g., "Signing in...") rather than getting confused by the SVG.
**Action:** Consistently include inline SVG loading spinners with `aria-hidden="true"` within buttons for asynchronous forms (e.g., login, register, submit) when `isLoading` is active, and center them with `flex items-center justify-center gap-2`.
