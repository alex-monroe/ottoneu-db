## 2024-02-27 - [Missing useMemo on Client-Side Text Filters]
**Learning:** Client-side components executing text filtering over array lists (like `PlayerSearch`) were missing `useMemo` hooks, leading to unnecessary O(N) recalculations on unrelated React re-renders. While `players.filter` works synchronously, as lists scale (e.g., hundreds of fantasy football players), executing string comparison methods (`toLowerCase().includes()`) inside every render cycle can accumulate significant main-thread latency, creating a performance bottleneck during component updates.
**Action:** When auditing or implementing client-side filtering on medium-to-large datasets, strictly enforce the use of `useMemo` with minimal dependencies (array, query string, filter enums) to isolate the recalculation cost.
